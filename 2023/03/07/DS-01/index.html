Definition

Recursive def of a tree:

A tree is a collection of N(N≥0)N(N \geq 0)N(N≥0) nodes and edges. The collection could be empty; otherwise the tree consists of a distinguished node rrr called root, and more than 000 nonempty (sub)trees T1,T2,⋯ ,TkT_1, T_2, \cdots,T_kT1​,T2​,⋯,Tk​, each of whose roots are connected from rrr by a directed edge.

  • parents/children: AAA is the parent of BBB and BBB is a child of AAA if there’s an edge from AAA to BBB.
  • siblings: AAA and BBB are siblings if their roots are same.
  • leaves: Node that has no children.
  • path (from n1n_1n1​ to nkn_knk​) and length: The length of the path defined as a sequence of n1,n2,⋯ ,nkn_1,n_2,\cdots,n_kn1​,n2​,⋯,nk​ such that nin_ini​ is parent of ni+1n_{i+1}ni+1​ for $1 \leq i\leq k $ is k−1k-1k−1.
  • depth/height (of nin_ini​): dep(ni)=len(path(root,ni)),height(ni)=max⁡l∈leaveslen(path(ni,l))dep(n_i) = len(path(root,n_i)), \quad height(n_i) = \max_{l \in leaves}len(path(n_i,l))dep(ni​)=len(path(root,ni​)),height(ni​)=maxl∈leaves​len(path(ni​,l))
  • depth/height (of tree TTT): dep(T)=max⁡l∈leavesdep(l)=height(T)=height(root))dep(T) = \max_{l \in leaves}dep(l) = height(T) = height(root))dep(T)=maxl∈leaves​dep(l)=height(T)=height(root))
  • ancestors/descendant : AAA is a ancestor of BBB and BBB is a descendant of AAA if there’s a path from AAA to BBB.

NOTE The height of an empty tree is defined -1 usually.

Average-Case Analysis

AVL Tree

AVL tree is a BST with a balance condition.

height balanced

  1. An empty binary tree is height balanced.
  2. If T is a nonempty binary tree with TLT_LTL​ and TRT_RTR​ as its left and right subtrees(could be empty), then T is height balanced iff
    • TLT_LTL​ and TRT_RTR​ are height balanced, and
    • ∣hL−hR∣≤1|h_L - h_R| \leq 1∣hL​−hR​∣≤1, where hLh_LhL​ and hRh_RhR​ are the heights of TLT_LTL​ and TRT_RTR​.

The balance factor (BF)

BF(node)=hL−hRBF(node) = h_L - h_R BF(node)=hL​−hR​

In an AVL tree, BF(node)∈−1,0,1BF(node) \in {-1,0,1}BF(node)∈−1,0,1.

Insertion and rotation

After an insertion, the nodes on the path from the insertion to the root may have their balance altered.

Hence after an insertion,

  1. follow the path up to the root and keep updating the balancing info until we met the first unbalanced node α\alphaα, i.e BF(node)∈Z−{−1,0,1}BF(node) \in \mathbb{Z} - \{-1,0,1\}BF(node)∈Z−{−1,0,1}.

  2. Address and fix the violation for different cases.

    Child Child’s Subtree Solution Note
    Left Left single rotation LL
    Left Right double rotation LR
    Right Left double rotation RL
    Right Right single rotation RR

  3. Repeat 1 - 2 until we gets to root.

Tree rotation: an operation on a binary tree that changes the structure without interfering with the order of the elements whose time complexity is O(1)O(1)O(1).

NOTEAfter a rotation, the side of the rotation increases its height by 1 while the side opposite the rotation decreases its height by 1 similarly.

  • LL Rotation: single right rotation is enough and no more need to rotate.

  • RR Rotation: single left rotation is enough and no more need to rotate.

  • LR Rotation: take k3k_3k3​ as the new root of the whole subtree by a left rotation and then a right rotation. This may cause new unbanlance.

  • RL Rotation: take k3k_3k3​ as the new root of the whole subtree by a right rotation and then a left rotation. This may cause new unbanlance.

Splay Tree

A splay tree is to control the time Any M consecutive tree operations starting from an empty tree taken at most O(Mlog⁡N)O(M\log N)O(MlogN) time.

Although this guarantee does not preclude the possibility that any single operation might take O(N)O(N)O(N) time.
But there are no bad input sequences.

Generally, when a sequence of MMM operations has total worst-case running time of O(MF(N))O(MF(N))O(MF(N)), we say that the amortized running time is O(F(N))O(F(N))O(F(N)).

Rotations

We will still rotate bottom up along the access path.
Let XXX be a nonroot node on the access path at which we are rotating.

  1. The parent of XXX is the root of the tree: merely rotate XXX and the root PPP.
  2. XXX has both a parent (PPP) and a grandparent (GGG):
    • zig-zag case: XXX is a right child and PPP is a left child (or vice versa). Exactly same as LR rotation or RL rotation of AVL tree.
    • zig-zig case: XXX and PPP are either both left children or both right children. take XXX as the new root of the whole subtree by a rotation between PPP and GGG and then a rotation between XXX and PPP.

Splaying not only moves the accessed node to the root, but also roughly halves the depth of most nodes on the path.

Amortized Analysis

ref Introduction to Algorithms, P451-462.

In an amortized analysis, we average the time required to perform a sequence of data-structure operations over all the operations performed.

  • Amortized analysis differs from average-case analysis in that probability is not involved;
  • amortized analysis guarantees the average performance of each operation in the worst case.

NOTE The amortized cost applies to each operation, even when there are several types of operations in the sequence.

Aggregate analysis

For all nnn, a sequence of nnn operations takes worst-case time T(n)T(n)T(n)in total. The amortized cost per operation is $ \frac{T(n)}{n}$.

Exp: MultiStack

  1. PUSH(S, x) pushes object x onto stack S.
  2. POP(S) pops the top of stack S and returns the popped object.
    err Calling POP on an empty stack.
  3. MULTIPOP(S, k) removes the k top objects of stack S.
    note popping the entire stack if the stack contains fewer than k objects; leaving the stack unchanged if k is not positive.

Then analyze a sequence of n PUSH,POP,and MULTIPOP operations on an initially empty stack. It’s easy to know that the worst case O(N)O(N)O(N) of MULTIPOP won’t exist continuously.

We can pop each object from the stack at most once for each time we have pushed it onto the stack. Therefore, the number of times that POP(including calls within MULTIPOP) can be called on a nonempty stack is at most the number of PUSH operations. Hence

T(n)=2n=O(n)T(n) = 2n = O(n) T(n)=2n=O(n)

T(n)n=O(1)\frac{T(n)}{n} = O(1) nT(n)​=O(1)
