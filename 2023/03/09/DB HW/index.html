3.Describe the circumstances in which you would choose to use embedded SQL rather than SQL alone or only a general-purpose programming language.

  • Since SQL does not provide the expressive power as full as a general-purpose language, there exist queries that can not be expressed in SQL, but can be expressed in a language such as C, Java, or Python. Under this kind of circumstances I would choose to embed SQL within a more powerful language to write such queries.
  • Nondeclarative actions cannot be done from within SQL. Applications usually have several components, and querying or updating data are only one of them. Other components are written in general purpose programming languages. Under this kind of circumstances I would choose to embed SQL within that language to combine them together.
