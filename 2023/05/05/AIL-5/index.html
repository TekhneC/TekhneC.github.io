AIL C5 知识图谱与描述逻辑

语义网络和知识图谱 可以用于表示特定领域中的结构化知识。

  • 本体 是一种支持知识共享的、统一的术语体系。
  • 描述逻辑 建模 本体。是语义网络和知识图谱推理的逻辑学基础。

语义网络和知识图谱

知识：

  • 在基于知识的系统中可以被表示为一组公式集合；
  • 通常具有一定结构。e.g 框架，网络。

在网络结构的知识体系中，知识在一个带标签的 有向图 中得到表达：

  • 图的结点：实体，可以是对象或概念。
  • 图的边：实体间关系
  • 两个节点和一条边的三元组：prop(Ind,Prop,Val)prop(Ind,Prop,Val)prop(Ind,Prop,Val)。
    • 结点IndIndInd，主语。
    • 边PropPropProp，动词，是一种特性。
    • 结点ValValVal，宾语。

语义网络

语义网络是一个图，结点表示对象或概念，弧表示这些对象或概念之间的关系。

知识图谱

知识图谱是一种语义网络，用于描述现实世界的各种实体及其关系。由 结点、边和标签组成。

  • 实体：对象，事件，情境，概念等
  • 核心：从大数据中提取结构化知识，以语义网络形式展示。

本体与描述逻辑

若要运用知识图谱中的各个元素进行推理，首先必须阐明这些元素及其关系的含义。在计算机领域，通常把符号的含义建构到文档中，即：在计算机中的符号与人脑中的概念之间*建立映射关系*。

  • 可以使用统一资源标识符 URI 来表示个体或特性。人们在使用URI时隐含了相关的含义。

  • 不同的建模者有不同的术语体系及其含义定义。通常需要建立一个统一的术语体系并阐明其中各个元素的含义来实现知识共享。

本体

统一的术语体系称为本体，可以被理解为一种用于描述特定领域 概念及其关系的形式化规范。

  • 关于事物分类的词汇表。
  • 分类的组织，e.g subClassOf或 subPropertyOf定义继承关系。
  • 一组公理集合，用于限制一些符号的定义以更好反映期望的含义。e.g 一些特性是传递的 / 论域和值域是受限的 / 特性值的数量是受限制的。

描述逻辑

描述逻辑是语义网络的继承和发展。它是一阶逻辑的子集，可以用于本体建模。目前有多种描述逻辑；每种描述逻辑都有自己的表示语言、表达能力和计算复杂性。

  • 属性语言 AL\cal{AL}AL：一种基本的语言，可以表示原子的否定、概念交、全称约束、受限的存 在量化。
  • 属性语言ALC\cal{ALC}ALC： 是一种代表性的语言，允许有任意的概念。

基本组成元素

描述逻辑中的三种实体:

  • 个体 - 一阶逻辑的个体常元。个体名称表示一个领域中的单个个体。
  • 概念 - 一阶逻辑的一元谓词。概念表示个体的集合。
  • 角色- 一阶逻辑的二元谓词。

公理

本体由一系列句子组成，这些句子被称为公理。

  • 公理必须在所描述的情况下为真。
  • 公理通常只描述本体所描述的情况的部分知识，可能存在与本体相一致的多种世界状态。即不同于数据库，本体不能完全描述特定的 “情况” 或 “世界的状态”.
  • 从逻辑的角度看，不同类型的公理之间没有主要区别。

断言型（ABox）公理：断言事实

ABox 公理可以刻画关于命名个体的知识：它们所属的概念以及它们相互之间的关系。

  • 概念断言

Mother(julia)julia:Mother \begin{align*} \rm Mother(julia)\\ \rm julia:Mother \end{align*} Mother(julia)julia:Mother​

  • 角色断言

ParentOf(julia,john) \rm ParentOf(julia,john) ParentOf(julia,john)

术语型 （TBox）公理：表达术语知识

TBox 公理描述概念之间的关系。

  • 概念包含

Mother⊑Parent \begin{align*} \rm Mother\sqsubseteq Parent \end{align*} Mother⊑Parent​

  • 概念等价 : 两个概念有相同的实例。

Person≡Human \rm Person \equiv Human Person≡Human

关系型（RBox）公理：建模角色间关系

RBox 公理是关于角色的特性的。

  • 角色包含公理

    ParentOf⊑ancestorOf\rm ParentOf \sqsubseteq ancestorOf ParentOf⊑ancestorOf

  • 角色等价公理

扩张

给定一个 TBox T\cal{T}T ，可将T\cal{T}T中的原子概念分为两个集合：

  • NT\cal{N_T}NT​:出现于公理左侧的名称符号集合；
  • BT\cal{B_T}BT​: 出现于公理右侧的基符号集合。

通常把名称符号称为被定义的概念，而把基符号称为原始概念。

  • 有环：T\cal{T}T中存在一个使用自身的原子概念。
  • 无环：T\cal{T}T中不存在一个使用自身的原子概念。

当一个 TBox 中的概念定义是无环的时，可通过一组原子概念定义所有其他概念，即将定义右侧的每个名称替换为该名称所代表的概念，通过一个最终停止的迭代过程得到一个术语 TBox T′\cal{T}'T′，称为T的扩张，它只包含形如 A≡C′A \equiv C'A≡C′ 的定义，其中 C′C'C′ 只包含基符号，不包含名称符号。

ALC\cal{ALC}ALC推理

概念推理

为一个领域建模时，可通过定义新概念来构建一个术语体系如T\cal{T}T。 此时需要厘清该新概念是否有意义或矛盾。

如果存在某个解释满足T\cal{T}T的公理（i.eT\cal{T}T的模型），使得该概念在该解释中表示一个非空集，那么该概念是有意义的。有意义的概念被称为关于T\cal{T}T可满足的。否则，称为关于T\cal{T}T不可满足的。

  • 可满足性：

    设T\cal{T}T是一个TBox。概念 CCC 是关于T\cal{T}T可满足的，如果存在T\cal{T}T的一个解释 I{\mathcal{I}}I，使得CIC^{\mathcal{I}}CI 非空。

可满足是概念的性质，概念间关系包括：包含、等价和不相交。

  • 概念包含：

    设T\cal{T}T是一个TBox。关于T\cal{T}T，概念CCC被包含于概念DDD，如果对于T\cal{T}T的每个解释I{\mathcal{I}}I，成立CI⊆DIC^{\mathcal{I}} \sube {\it D}^ICI⊆DI。记CCC包含于DDD 为 T⊨C⊑D{\cal{T}} \models C \sqsubseteq DT⊨C⊑D。

  • 概念等价：

    设T\cal{T}T是一个TBox。概念CCC与DDD关于T\cal{T}T等价，如果对于T\cal{T}T的每个解释I\cal{I}I，CI=DIC^{\cal{I}} = D^{\cal{I}}CI=DI， 记作T⊨C≡D{\cal{T}} \models C \equiv DT⊨C≡D。

  • 概念不相交：
    设T\cal{T}T是一个TBox。概念CCC与DDD关于T\cal{T}T不相交，如果对于T\cal{T}T的每个解释 I\cal{I}I，CI∩DI=ϕC^{\mathcal{I}} \cap {\it D}^I = \phiCI∩DI=ϕ。

  • 与规则 / ⊓\sqcap⊓ 规则：如果x:(C⊓D)∈Ax : (C\sqcap D) \in\cal Ax:(C⊓D)∈A，且{x:C,x:D}⊈A\{x : C, x : D\} \not\sube\cal A{x:C,x:D}⊆A，那么A′=A∪{x:C,x:D}{\cal A}' = {\cal A}∪\left\{x : C, x : D\right\}A′=A∪{x:C,x:D}。
  • 或规则 / ⊔\sqcup⊔ 规则：如果x:(C⊔D)∈Ax : (C\sqcup D) \in\cal Ax:(C⊔D)∈A，且{x:C,x:D}∩A=ϕ\{x : C, x : D\} \cap {\cal A} = \phi{x:C,x:D}∩A=ϕ，那么A′=A∪{x:C},A′′=A∪{x:D}{\cal A}' = {\cal A}∪\left\{x : C\right\},{\cal A}'' = {\cal A}∪\left\{x : D\right\}A′=A∪{x:C},A′′=A∪{x:D}。
  • 存在 / ∃\exist∃ 规则：如果x:∃R.C∈Ax:\exist R.C \in \cal Ax:∃R.C∈A，而且不存在 zzz ，z:C∈Az : C \in Az:C∈A且(x,z):R∈A(x, z) : R \in\cal A(x,z):R∈A，则对于一个新的个体 y∈A,A′=A∪{y:C,(x,y):R}y\in{\cal A, A'} = {\cal A} \cup \left\{y : C,(x, y) : R\right\}y∈A,A′=A∪{y:C,(x,y):R}。
  • 全称 / ∀\forall∀ 规则：如果x:∀R.C∈Ax:\forall R.C \in \cal Ax:∀R.C∈A且(x,y):R∈A(x,y):R \in \cal A(x,y):R∈A，但，y:C∉Ay : C \not\in Ay:C∈A且(x,z):R∈A(x, z) : R \in\cal A(x,z):R∈A，则 A′=A∪{y:C}{\cal A'} = {\cal A} \cup \left\{y : C\right\}A′=A∪{y:C}。
ABox的性质
  • ABox的冲突性：
    ABoxA\cal AA包含冲突 / 冲突的，若存在个体名 aaa ，概念CCC使得{a:C,a:¬C}⊆A\{a: C, a : \lnot C\} \sube \cal A{a:C,a:¬C}⊆A， 或者⊥⊆A\bot \sube \cal A⊥⊆A。否则称为 无冲突的。

算法从ABox A0={x0:C0}{\cal A_0} = \{x_0 : C_0\}A0​={x0​:C0​} 开始，不断应用表转换规则直到没有更多的规则可应用。

  • ABox的完全性：

    A\cal AA是完全的，如果A\cal AA 包含冲突，或没有一个扩张规则可应用的。

  • ABox的一致性：

    A0\cal A_0A0​是一致的，如果不断应用扩张规则得到的完全的A\cal AA 是无冲突的。C0C_0C0​也是可满足的。
    否则称为不一致的，C0C_0C0​是不可满足的。
