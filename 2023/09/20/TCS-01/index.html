Problems

Take Minimum spanning tree, MST as instance.

  • Optimization problem: Given G=(V,E,W)G = (V,E,W)G=(V,E,W), find out MST.
  • Search problem: Given G=(V,E,W),k∈Z+G = (V,E,W), k\in \mathbb{Z}+G=(V,E,W),k∈Z+, find out 1 MST whose weight-sum is smaller than kkk.
  • Decision problem: Given G=(V,E,W),k∈Z+G = (V,E,W),k \in \mathbb{Z}+G=(V,E,W),k∈Z+, find out whether there is a MST whose weight-sum is smaller than kkk.
  • Counting problem: Given G=(V,E,W),k∈Z+G = (V,E,W),k \in \mathbb{Z}+G=(V,E,W),k∈Z+, find out the number of MSTs whose weight-sums are smaller than kkk.

It’s obvious that Optimization / Search / Counting problem is harder than Decision problem.

That’s say, if Decision is hard, then the other three is also hard.

Take Decision problem to be considered.

  • problem - instance
    • yes-instance
    • no-instance

For our machine, the input G,kG,kG,k is encoded. Then the problem could be abstracted into

Given the encodings of G,KG,KG,K, whether the code is an element of the set of the yes-instance?

A problem is uniquely decided by the set of the yes-instance, which is a Language.

Decision Problem⇔Language\mathrm{Decision\ Problem} \Leftrightarrow \mathrm{Language} Decision Problem⇔Language

  1. Decision Problem⇒encodings of yes−instances⇒Language\mathrm{Decision\ Problem} \Rightarrow \mathrm{encodings\ of\ yes-instances}\Rightarrow\mathrm{Language}Decision Problem⇒encodings of yes−instances⇒Language
  2. $\mathrm{Language} \Rightarrow \mathrm{Decision\ Problem}:\small Given\ string\ w, is\ w \in L? $
